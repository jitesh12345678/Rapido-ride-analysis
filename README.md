# Rapido Ride Analysis Dashboard

## Project Overview

This project analyzes 50,000 Rapido ride records to uncover insights related to ride demand, revenue generation, customer behavior, payment preferences, and ride cancellations. The dataset was cleaned and analyzed using SQL, and the results were visualized through an interactive Power BI dashboard.

## Objectives

* Analyze ride performance across different service types.
* Measure total revenue and ride volume.
* Identify cancellation trends.
* Understand customer payment preferences.
* Discover high-demand pickup locations.
* Evaluate ride distance and duration patterns.

## Dataset Information

The dataset contains 50,000 ride records and 13 attributes:

* Services
* Date
* Time
* Ride Status
* Source
* Destination
* Duration
* Ride ID
* Distance
* Ride Charge
* Misc Charge
* Total Fare
* Payment Method

## Data Cleaning

The following data cleaning steps were performed:

* Checked for missing values.
* Handled null values in fare-related columns.
* Validated data types for date, distance, and fare fields.
* Checked for duplicate records.
* Standardized categorical values.
* Verified data consistency across ride status and payment methods.

## SQL Analysis

SQL was used to perform:

* Completed ride analysis
* Revenue calculations
* Average trip distance analysis
* Service-wise performance comparison
* Cancellation analysis
* Payment method analysis
* Top pickup location identification
* Longest ride analysis

## Dashboard KPIs

The Power BI dashboard includes:

* Total Rides
* Total Revenue
* Average Fare
* Average Distance
* Cancellation Rate
* Revenue by Service Type
* Payment Method Distribution
* Ride Status Analysis
* Location-Based Demand Analysis

## Key Insights

* Revenue contribution varies across service categories.
* Certain pickup locations generate significantly higher ride demand.
* Digital payment methods are widely used by customers.
* Ride cancellations impact overall operational efficiency.
* Distance and duration directly influence fare generation.

## Tools Used

* SQL

## Conclusion

This project demonstrates an end-to-end data analytics workflow, including data cleaning, SQL-based analysis, KPI development, and dashboard visualization. The dashboard provides actionable insights that can help improve operational efficiency and support data-driven decision-making.
